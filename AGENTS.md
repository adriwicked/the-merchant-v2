# The Merchant v2

A 2D tile-based game built with **Phaser 3**, **TypeScript**, and **Vite**. Deployed to GitHub Pages via GitHub Actions on every push to `main`.

## Tech Stack

- **Phaser 3** — game engine (canvas renderer)
- **TypeScript** — strict types, no `any`
- **Vite** — dev server and build (`base: '/the-merchant-v2/'`)
- **simplex-noise** — procedural terrain generation
- **GitHub Actions** — auto-deploy to GitHub Pages (`.github/workflows/deploy.yml`)

## Project Structure

```
src/
  config.ts        — Grid constants, color palette, shared helpers (tweakColor, desaturate, brightenColor, cellPosition)
  index.ts         — Phaser config + GameScene (world map rendering, water animation, location hover/click, iris-out transition)
  map.ts           — Terrain enum (9 types), noise generation, radial falloff, terrain classification
  locations.ts     — LocationType enum, GameLocation interface, placement logic with distance constraints
  game-state.ts    — Persistent world state (WorldState, MineState), generated once and cached in module scope
  mine-scene.ts    — MineScene class, MineTile enum, cavern generation, player movement, bombs, gold nuggets, torch lighting
  inventory.ts     — Global inventory object (bombs, goldNuggets), persists across scenes
  debug-panel.ts   — Debug UI panel (exists but NOT imported — hidden, kept for future use)
```

## Grid System

All rendering is tile-based on a 45x45 grid:

- `CELL_SIZE: 10` — pixel size of each tile
- `CELL_SEPARATION: 2` — gap between tiles AND between tiles and the border
- `BORDER_WIDTH: 3` — outer border thickness
- `CANVAS_WIDTH/HEIGHT: 600` — fixed canvas, no scaling (`Phaser.Scale.NONE` + `CENTER_BOTH`)
- `GRID_PIXEL_WIDTH = MAP_WIDTH * (CELL_SIZE + CELL_SEPARATION) + CELL_SEPARATION`

Use `cellPosition(row, col)` from `config.ts` for all pixel coordinate calculations.

## State Architecture

**Critical pattern**: Phaser's `scene.start()` destroys all game objects. Persistent data lives in plain TypeScript modules, NOT on scene instances.

- `game-state.ts` — `getWorldState()` returns a singleton. World map, locations, water colors generated once. `getMineState(loc)` returns per-mine persistent state (grid, tile colors, nugget positions).
- `inventory.ts` — Global `{ bombs, goldNuggets }` object, survives scene transitions.
- Visual Phaser objects (Graphics, Rectangles, Text) are recreated from cached state each time a scene starts.

## Key Conventions

### Color Manipulation
All colors are `0xRRGGBB` numbers manipulated via bit shifting. Three helpers in `config.ts`:
- `tweakColor(color)` — slight random brightness/saturation variation per tile
- `desaturate(color, amount)` — blend toward gray (0-1)
- `brightenColor(color, factor, warmth)` — multiply brightness with optional warm tint

### Terrain Generation
1. Generate raw noise grid with `simplex-noise` (fractal, 4 octaves)
2. Apply radial falloff as post-process (inner 1/3 radius safe, linear ramp to edges)
3. Classify terrain by thresholds (9 types: DEEP_WATER through ROCK)
4. Detect FOAM (water adjacent to beach)

### Animations
Water and torch flicker use the same pattern: dedicated `Phaser.GameObjects.Graphics` layer, base colors stored in state, re-tweaked every 2 seconds via `this.time.addEvent`.

### Locations
- Cities: red (`0xdd3333`), placed on LOW_GRASS adjacent to BEACH_SAND
- Mines: yellow (`0xccaa44`), placed on DIRT or ROCK
- Max 3 cities, max 2 mines. Min distance constraints between all locations.
- Hover: desaturated normal state -> full saturation + expanded square on hover

### Mine Interior
- Irregular cavern generated by flood-fill expansion (~10 tile radius)
- MineTile enum: BLACK(0), WALL(1), FLOOR(2), TORCH(3)
- Very dark colors (WALL `0x382a1a`, FLOOR `0x3d332a`) for contrast with torch light
- Torch lighting: radius 6, brightness 1.5, warmth 0.6, linear falloff, strongest-wins (no stacking)
- Bombs: space to plant, 2s fuse, irregular explosion radius 3-5, 15% gold nugget spawn per converted tile
- Post-explosion: wall regeneration pass (BLACK tiles adjacent to FLOOR become WALL), torch recalculation

## Commands

```bash
npm run dev      # Start Vite dev server
npm run build    # Production build to dist/
```

## Deployment

Pushes to `main` trigger `.github/workflows/deploy.yml` which builds and deploys to GitHub Pages. The `package-lock.json` must be committed for CI to work.

## Things to Watch Out For

- **No circular imports**: `index.ts` imports scene files; shared code goes in `config.ts`
- **Phaser Rectangle positioning**: positioned by center, use even pixel sizes to avoid subpixel rendering
- **Foam pass**: collect foam positions first, then apply — avoids cascading adjacency detection
- **Iris-out**: must cover CELL_SEPARATION gaps (paint darkened cells with padding) and hide location visuals at start
- **Mine state**: `MineState.grid` stores `number[][]` (MineTile enum values), not objects. Nugget positions are a `Set<string>` with `"row,col"` keys.
